# SGI Operational Research Template (Post-Ω Closure)

## Non-Negotiable Structure

Every future SGI output MUST follow this 7-component structure:

---

### 1. SYSTEM

Name one concrete system only.

Examples:
- Kubernetes cluster
- ant colony
- repeated-game social network
- immune signaling network
- swarm robotics platform

No cross-domain discussion yet.

### 2. PERSISTENCE OBJECT

Define exactly what persists.

Examples:
- task allocation topology
- communication backbone
- covariance eigenspace
- role specialization
- routing structure
- recovery trajectory

Must be operationally measurable.

### 3. PERTURBATION

Specify exact disturbance.

Examples:
- 30% node failure
- delayed communication
- resource starvation
- mutation injection
- adversarial rewiring

No vague "dynamic stress."

### 4. OBSERVABLES

List measurable quantities only.

Examples:
- recovery time
- spectral radius
- coordination entropy
- eigenspace drift
- routing variance
- cluster persistence duration

No interpretive language.

### 5. REPRESENTATION AUDIT

Mandatory. For every metric:
- what changes under embedding?
- what survives normalization?
- what depends on segmentation?
- what is gauge-removable?

### 6. FAILURE CONDITION

Mandatory pre-registration.

Examples:
- "If persistence disappears after normalization, the structure was gauge-induced."
- "If cluster stability depends entirely on observer segmentation, no transportable organization has been identified."

### 7. CROSS-DOMAIN TRANSFER

ONLY after the single-system study succeeds.

Ask:
- does the same persistence grammar recur elsewhere?
- does the same recovery law appear?
- does the same gauge-removable structure appear?
